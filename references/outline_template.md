# 通用 PPT 大纲模板（深度分析报告）

本模板由 `ppt_大纲.md` 抽象而来，适配任意主题的"深度分析报告"幻灯片产出。
模板中所有 `<...>` 形式的占位符在生成阶段都会被主题专属内容替换，并最终落地为
`out/<主题>深度分析ppt生成prompt.md`，作为后续 HTML / imagegen 渲染的统一提示词。

> 默认输出语言：**简体中文**；引文与一手术语保留原语种并括注译名。
> 默认篇幅：**25 页**（可在 12–35 之间根据主题伸缩）。
> 默认版式：16:9。

---

## 全局风格（Global Prompt）

**视觉风格**：默认主色 `<PRIMARY 例：科技蓝 #0B2545>`，强调色 `<ACCENT 例：青蓝 #43B0F1>`，
中性背景 `<NEUTRAL 例：极简白 #F5F7FA>`，对比色用于"安全/警示"类语义，例如
`<POSITIVE 例：盾牌绿 #2BB673>` / `<WARN 例：警示橙红 #FF6B6B>`。

**风格基调**："扁平化 + 微拟物"，多用节点网络、容器/沙箱、代码框、流程箭头、雷达/矩阵等元素；
科技感、专业感、克制留白。

**排版原则**：字不如表，表不如图。每页一条主张；关键数据加粗高亮；保持留白；避免文字堆砌。

**字体规范**：标题 36pt 粗体（中文：思源黑体 Heavy / 苹方 Bold；英文：Inter / Söhne）；
正文 18pt 常规；脚注 10pt 60% 不透明度。

---

## 章节骨架

> 默认三段式：**趋势分析 → 趋势判断 → 趋势动作**；可按主题改写，但建议保留"看世界 → 看自己 → 怎么干"的三幕结构。

| 段 | 名称 | 子页数 | 目的 |
|---|---|---|---|
| 0 | 封面 + 目录 | 2 | 统一开场、建立信息架构 |
| Ⅰ | `<段一名 默认：技术趋势分析>` | 6–8 | 看大势、看行业、看对手、看客户 |
| Ⅱ | `<段二名 默认：技术趋势判断>` | 6–9 | 是什么、做什么、带来什么、关键技术、成熟度、产业链、预判 |
| Ⅲ | `<段三名 默认：技术趋势动作>` | 5–7 | 价值定位、落地方案、子课题、里程碑、总结 |

---

## 逐页模板（占位符版）

每一页都遵循以下结构填写；生成阶段会将占位符替换为主题专属表述。

```
幻灯片 N：<页面名>
- 角色（archetype）：cover | agenda | section-divider | definition | landscape |
                     architecture | timeline | matrix | data-point | quote |
                     comparison | challenges | recommendations | references
- 布局：<例：左标题右插画 / 顶部标题+三栏卡片 / 中央矩阵+四角说明>
- 视觉描述：<给 imagegen 的画面描述；HTML 渲染时作为背景插画提示>
- 内容补充细化：
  - 主标题：<主标题>
  - 副标题：<副标题（可空）>
  - 关键要点（≤5 条）：
    1. <bullet 1，≤25 字>
    2. <bullet 2>
    3. <bullet 3>
  - 数据/引文：<可选的数字、引言或来源标签>
- 演讲稿（speaker_notes）：<60–150 字，必须标注来源 [raw/<file>.md]>
```

### Ⅰ. 封面与目录

#### 幻灯片 1：封面页
- 角色：cover（hero）
- 布局：居中标题大字号，下方副标题与日期；左下角 logo 占位。
- 视觉描述：`<深色科技背景，发光的分布式网络节点连线，象征 <主题英文关键词>>`。
- 内容补充细化：
  - 主标题：**`<主题>`深度分析报告**
  - 副标题：`<一句话主张，例：从 X 到 Y 的演进与落地路径>`
  - 汇报人：`<团队/个人>`
  - 时间：`<YYYY.MM>`

#### 幻灯片 2：目录页
- 角色：agenda
- 布局：三栏卡片，每栏对应一段；每栏列 3–4 个子标题。
- 视觉描述：极简白底，三色卡片对应三段；卡片右下角带阶段编号。
- 内容补充细化：列出三段标题及各段核心子页。

### Ⅱ. 第一部分：`<段一名 默认：技术趋势分析>`

#### 幻灯片 3：技术大背景
- 角色：definition / data-point
- 布局：左侧大字号关键论断，右侧三组数据/曲线。
- 视觉描述：`<示意图：背景为 <主题> 全景；三个关键指标以大字数字呈现>`。
- 内容补充细化：定义 + 历史拐点 + 当前规模/增速。

#### 幻灯片 4：看行业（标准化趋势）
- 角色：timeline / landscape
- 布局：水平时间线 + 三个标准化组织节点。
- 视觉描述：`<横向时间线，节点列出 <SDO1/SDO2/SDO3>，年份在上、动作在下>`。
- 内容补充细化：列出主要标准组织（如 3GPP/ETSI/TM Forum/IEEE/ITU-T/CCSA）的近 2 年立项与里程碑。

#### 幻灯片 5：看对手（开源生态）
- 角色：landscape / matrix
- 布局：四象限矩阵或卡片墙，覆盖头部 6–10 个开源项目。
- 视觉描述：`<开源项目 logo 占位 + 简要标签：协议、首次发布、Stars 量级>`。
- 内容补充细化：每个项目一行卖点 + 一行风险。

#### 幻灯片 6：看对手（海外巨头）
- 角色：comparison
- 布局：表格 / 卡片对比，3–5 家厂商。
- 视觉描述：`<厂商徽标占位 + 旗舰产品名 + 关键能力标签>`。
- 内容补充细化：每家一句话定位 + 一项最强能力 + 一项软肋。

#### 幻灯片 7：看对手（国内大厂）
- 角色：comparison
- 同上，覆盖国内 4–6 家代表企业。

#### 幻灯片 8：看客户（需求变化）
- 角色：data-point / quote
- 布局：左侧客户语录卡，右侧三条需求趋势曲线。
- 视觉描述：`<分行业卡片：金融 / 政企 / 运营商 / 制造，每张卡片一条客户原话>`。
- 内容补充细化：客户由"试点验证"向"规模复制"迁移的 3 个信号。

#### 幻灯片 9：第一阶段论点（趋势断言）
- 角色：recommendations
- 布局：三条主张 + 每条一句证据。
- 视觉描述：`<三条横向条带，每条一个图标 + 主张>`。
- 内容补充细化：把第一部分的所有信号收敛成 3 条断言。

## 把通用大纲细化为"主题专属 prompt"

在生成阶段，必须根据本通用大纲产出 `out/<主题>深度分析ppt生成prompt.md`，
并用它驱动 HTML 或 imagegen 渲染。最小输出格式：

````markdown
# <主题>深度分析 PPT 生成提示词

## 全局风格
- 主色 / 强调色 / 中性 / 对比色
- 字体规范
- 风格基调（要写成可直接喂给图像/HTML 渲染器的英文短语）

## 渲染模式
- mode: <html | imagegen>
- 输出语言: zh-CN（默认）
- 篇幅: <N> 页

## 逐页提示词
### 幻灯片 1：<页面名>
- archetype: <cover | ...>
- 布局: <...>
- 视觉描述（中文）: <...>
- visual_prompt（英文，喂给 imagegen 或 HTML 背景渲染）: <...>
- 主标题 / 副标题 / 关键要点 / 数据 / 引文
- speaker_notes: <...>，来源: [raw/<file>.md]

### 幻灯片 2：...
````

> 这个 `prompt.md` 与 `analysis/slides.json` 是**等价的人类可读版**：每次手工调整这份
> 文件后，需要同步回写 `slides.json`，由 Stage 4/5 的脚本读取。

## 校验清单（生成前必须通过）

- [ ] 每一页都有 archetype、布局、视觉描述、视觉 prompt（英文）
- [ ] 每一页都有简体中文标题与要点（除非引文页保留原语种）
- [ ] 每条要点都能溯源到 `raw/<NN>-<slug>.md` 中的某一条证据
- [ ] 全文档使用同一套调色板与字体
- [ ] 任意相邻两页的 archetype 不重复（避免连续 wall-of-bullets）
- [ ] 篇幅在 12–35 页区间，目标默认 25 页
