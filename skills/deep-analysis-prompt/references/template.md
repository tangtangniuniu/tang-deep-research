# 深度行业技术分析报告 - 通用搜索提示词生成模板

## 使用说明

本文件是一个**元提示词（Meta-Prompt）**：将用户输入的研究主题填入下方变量，即可生成一份用于深度搜索分析的完整提示词，最终输出可直接用于驱动生成高质量、专业科技感的演示文稿（PPT）。

### 输入变量（请按需替换）

| 变量 | 说明 | 示例 |
|---|---|---|
| `{{TOPIC}}` | 研究主题（核心技术 / 行业 / 产品方向） | 大模型Agent技术 / AI编码助手 / 6G通信 / 算力网络 |
| `{{YEAR}}` | 报告年份（默认当前年） | 2026 |
| `{{REPORT_MONTH}}` | 报告月份 | 2026.03 |
| `{{INDUSTRY_PLAYERS}}` | 行业代表厂商列表（国际 + 开源 + 中国） | google、openai、anthropic、deepseek、智谱… |
| `{{COMPETITORS}}` | 同业竞争对手 | 华为、思科、爱立信、烽火、腾讯、阿里、百度… |
| `{{REGIONS}}` | 政策区域 | 美国、中国、欧盟 |
| `{{CUSTOMERS}}` | 关键客户/行业用户 | 移动、联通、电信、国外电信运营商… |
| `{{SELF}}` | 自己/本公司 | 中兴 / xxx |
| `{{SUBTOPIC_1}}` | 子课题1（落地场景1） | 研发闭环 |
| `{{SUBTOPIC_2}}` | 子课题2（落地场景2） | 网络故障定界自愈 |
| `{{KEY_TECHS}}` | 关键技术清单（与主题相关） | agentic、agent harness、skills、memory、上下文管理… |
| `{{KEY_PRODUCTS}}` | 主题下需重点关注的产品/项目 | openclaw、claude code、hermes agent… |
| `{{ANALYSTS}}` | 行业分析机构 | 麦肯锡、Gartner、IDC、Forrester… |

---

## 通用约束（所有主题通用）

> 以下规则在生成的提示词中必须始终保留，作为"硬约束"。

1. **资料新鲜度**：仅使用 `{{YEAR}}` 当年与上一年（最近 12 个月）公开材料；**严禁引用 `{{YEAR}}-2` 及更早年份的过期资料**作为主要论据。
2. **信息密度**：每张幻灯片信息点高度密集，避免概括性语言；必须列出**具体厂商 / 具体产品 / 具体白皮书 / 具体演示材料 / 具体版本号 / 具体发布会**。
3. **专业精深**：思路清晰、内容详细、有条理，**禁止空话套话**，每一页都要可验证、可追溯。
4. **避免重复**：不同章节之间不重复同一论据；同一章节内部条目互斥、各有侧重。
5. **突出关注点**：明确标记"令人感兴趣的点 / 反共识发现 / 业界争议点"，让读者抓住重点。
6. **引用可追溯**：尽量给出原始链接、白皮书名、会议/演讲标题、发布日期。

## 元提示词使用方法（给生成代理 / LLM 的指令）

当用户输入研究主题时，按以下步骤生成最终搜索/分析提示词：

1. **解析主题** `{{TOPIC}}`：识别其所属领域（AI / 通信 / 芯片 / 软件…），自动推断默认变量。
2. **变量自动补全**：
   - `{{YEAR}}` 默认取**当前年份**；`{{REPORT_MONTH}}` 默认取**当前年.月**。
   - `{{INDUSTRY_PLAYERS}}` 根据主题领域，自动列出 6-10 家代表性国际/开源/中国厂商。
   - `{{COMPETITORS}}` 根据 `{{SELF}}` 所在赛道，自动列出 4-8 家直接对手。
   - `{{KEY_PRODUCTS}}` 必须为 `{{YEAR}}` 当年活跃的产品/项目；`{{KEY_TECHS}}` 必须是当下技术栈热词。
   - `{{ANALYSTS}}` 默认 `麦肯锡、Gartner、IDC、Forrester、CB Insights、信通院`。
3. **缺失变量**：若用户未提供 `{{SELF}}` / `{{SUBTOPIC_*}}` / `{{CUSTOMERS}}`，**主动询问用户**，不要自行编造。
4. **输出格式**：将填好变量的"提示词生成模板"完整输出，作为下游深度研究 / PPT 生成代理的输入。
5. **强制约束**：始终保留"通用约束"中 6 条规则，并在输出末尾以"### 硬性要求"的形式重申。
