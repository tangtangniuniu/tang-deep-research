# Deep Analysis Prompt Generator

把"研究主题"转成一份**高质量深度分析报告提示词**——可直接交给下游 LLM / 深度研究 / PPT 生成代理使用。

> 默认输出语言：**简体中文**。
> 本 skill **只生成提示词**，不做联网检索，不做 PPT 渲染（那是 `tang-deep-research` 的职责）。

---

## 何时触发

- "帮我生成一个 XX 主题的深度分析提示词 / search prompt / 调研 prompt"
- "为 XX 写一份行业分析的提示词模板"
- "我想让模型对 XX 做深度行业分析，先帮我把 prompt 写好"
- "把这个研究主题套到深度分析模板里"

不触发：用户已经直接要 PPT 或调研结果时——那走 `tang-deep-research` 全流水线。

---

## 输入契约（13 个变量）

| 变量 | 必须？ | 默认 / 推断规则 |
|---|---|---|
| `{{TOPIC}}` | ✅ 必须 | 从用户消息抽取；抽不到就追问 |
| `{{YEAR}}` | 自动 | 当前年份（如 2026） |
| `{{REPORT_MONTH}}` | 自动 | 当前年.月（如 2026.05） |
| `{{SELF}}` | ✅ 必须 | 询问"本公司/委托方是谁"，**不要编造** |
| `{{SUBTOPIC_1}}` `{{SUBTOPIC_2}}` | ✅ 必须 | 询问两个子课题/落地场景 |
| `{{CUSTOMERS}}` | 询问 | 关键客户/行业用户 |
| `{{INDUSTRY_PLAYERS}}` | 自动可补 | 根据主题自动列 6-10 家国际+开源+中国厂商，让用户增删 |
| `{{COMPETITORS}}` | 自动可补 | 根据 `SELF` 所在赛道列 4-8 家直接对手 |
| `{{KEY_PRODUCTS}}` | 自动可补 | 当年活跃产品/项目 |
| `{{KEY_TECHS}}` | 自动可补 | 主题的技术栈热词 |
| `{{REGIONS}}` | 默认 | 美国、中国、欧盟 |
| `{{ANALYSTS}}` | 默认 | 麦肯锡、Gartner、IDC、Forrester、CB Insights、信通院 |

**铁律**：`SELF` / `SUBTOPIC_*` / `CUSTOMERS` 不允许 LLM 自行编造——必须问用户。其它变量可以先给出推断列表，再让用户增删。

---

## 流程（4 步）

### Step 1 — 抽取主题、确认变量

读用户消息，拿到 `TOPIC`。然后**一次性发问**收齐缺失的"必须"变量：

> 我准备生成『<TOPIC>』的深度分析提示词。还需要确认 3 件事：
> 1) 本公司 / 委托方是谁？（影响『看自己』那一页）
> 2) 想重点展开哪两个落地子课题？
> 3) 关键客户/目标用户是谁？
>
> 另外我会自动补全这些变量，不合适请直接改：
> - 行业代表厂商：…（基于主题自动列 6-10 家）
> - 直接竞争对手：…（基于本公司自动列 4-8 家）
> - 关键产品：…
> - 关键技术：…

把回答连同自动补全结果，写到 `run.config.json`：

```json
{
  "topic": "...",
  "year": 2026,
  "report_month": "2026.05",
  "self": "...",
  "subtopic_1": "...",
  "subtopic_2": "...",
  "customers": "...",
  "industry_players": ["...", "..."],
  "competitors": ["...", "..."],
  "key_products": ["...", "..."],
  "key_techs": ["...", "..."],
  "regions": ["美国", "中国", "欧盟"],
  "analysts": ["麦肯锡", "Gartner", "IDC", "Forrester", "CB Insights", "信通院"]
}
```

### Step 2 — 套模板

读取 `references/template.md`（**只读，不修改**），把所有 `{{...}}` 占位符**逐个**替换为 `run.config.json` 里的值。规则：

- 列表型变量（`INDUSTRY_PLAYERS` 等）用顿号串接，末尾加"等"；
- `{{YEAR+1}}` `{{YEAR+2}}` `{{YEAR-1}}` 按算术计算（如 2026 → 2027 / 2028 / 2025）；
- 替换后**全文 grep `{{`**确认没有残留占位符；有就回 Step 1 补问。

### Step 3 — 注入硬性要求

在生成的提示词末尾追加固定的 `### 硬性要求` 段（来自 template.md 末尾），保留原文，不允许改写。

### Step 4 — 输出与回显

写到 `out/<TOPIC>-prompt.md`（如 `out/AI编码助手-prompt.md`）。

回显给用户：
1. 文件路径
2. 提示词的前 30 行（让用户快速判断风格是否对）
3. 一句"是否需要调整变量重新生成"

## 通用约束（必须保留在生成的提示词里）

> 这 6 条来自 `references/template.md`，写进每一份生成物的"硬性要求"段，**不可删改**：

1. 资料新鲜度：仅最近 12 个月（`{{YEAR-1}} ~ {{YEAR}}`），禁用 `{{YEAR}}-2` 及更早材料作为主论据。
2. 信息密度：必须列具体厂商、产品名、版本号、白皮书/演讲/发布会名称。
3. 专业精深：禁止"赋能/抓手/闭环/生态"等空话；用数据和案例代替形容词。
4. 论据互斥：同一论据不在两处出现。
5. 关注点显式标注：每页 2-3 条【关注点】或【有趣发现】，突出反共识或业界争议。
6. 引用可追溯：尽量给出原文链接、发布日期。

---

## 反模式（不要做）

- ❌ 没问 `SELF` / `SUBTOPIC_*` / `CUSTOMERS` 就编造一个填进去
- ❌ 把 `KEY_PRODUCTS` 填成 `{{YEAR}}-2` 之前就停更的产品（如已被取代的旧版）
- ❌ 直接修改 `references/template.md` 模板原文
- ❌ 越界去做联网检索或写 PPT —— 那是 `tang-deep-research` 的活
- ❌ 把生成的提示词写成"摘要 / 缩略版"——必须是完整可运行的提示词

## 文件清单

- `SKILL.md` — 本文
- `references/template.md` — 通用 25 页大纲模板（占位符版，**只读**）
- `out/<主题>-prompt.md` — 每次运行的产物

## 与 `tang-deep-research` 的关系

| | deep-analysis-prompt | tang-deep-research |
|---|---|---|
| 范围 | 只生成提示词 | 提示词 → 素材 → 分析 → PPT 全流水线 |
| 触发 | "帮我写个 prompt" | "帮我做个 PPT / 调研报告" |
| 产物 | `out/<主题>-prompt.md` | `out/<主题>.pptx` |
| 依赖 | 无 | python-pptx / playwright / imagegen |

可以把本 skill 当作 `tang-deep-research` Stage 3 的独立可调用版本——只想要提示词、不想跑后面渲染流水线时用它。
